k8s란?

**Kubernetes (k8s)**는 컨테이너화된 애플리케이션을 자동으로 배포, 확장, 관리해주는 오픈소스 컨테이너 오케스트레이션 플랫폼입니다. 구글에서 개발한 기술을 기반으로 CNCF(Cloud Native Computing Foundation)에 기부되었고, 현재는 클라우드 환경에서 가장 널리 사용되는 컨테이너 관리 시스템이에요.

핵심 개념 정리

  • 컨테이너(Container): 애플리케이션과 그 실행 환경을 패키징한 단위 (주로 Docker 사용)
  • Pod: Kubernetes의 가장 작은 배포 단위. 하나 이상의 컨테이너를 포함할 수 있어요.
  • Node: 컨테이너가 실제로 실행되는 물리/가상 머신
  • Cluster: 여러 노드로 구성된 Kubernetes의 전체 인프라
  • Deployment: 애플리케이션의 원하는 상태(예: 몇 개의 복제본 유지 등)를 정의하고 관리
  • Service: Pod에 접근할 수 있는 안정적인 네트워크 주소를 제공 (LoadBalancer, ClusterIP, NodePort 등)
  • Ingress: 외부 트래픽을 내부 서비스로 라우팅해주는 HTTP(S) 레벨 라우터
  • ConfigMap / Secret: 애플리케이션 설정 값이나 민감한 정보를 외부에서 주입

Kubernetes의 장점

  • 자동화된 배포 및 롤백: 애플리케이션을 손쉽게 배포/업데이트하고, 문제 발생 시 자동으로 이전 버전으로 롤백 가능
  • 자동 스케일링: 트래픽에 따라 Pod 수를 자동 조절 가능
  • 자가 치유(Self-healing): 장애 난 컨테이너는 자동으로 다시 생성
  • 서비스 디스커버리 및 로드 밸런싱: Pod 간 통신을 쉽게 할 수 있게 도와줌

간단한 동작 흐름

  1. 사용자는 애플리케이션을 컨테이너(Docker)로 패키징
  2. 이를 Kubernetes 클러스터에 배포
  3. Kubernetes가 이를 적절한 Node에 스케줄링하고 실행
  4. 서비스나 Ingress를 통해 외부 접근 가능

helm 이란?

Helm은 Kubernetes의 패키지 관리자입니다. 쉽게 말해서, Kubernetes에서 애플리케이션을 더 쉽고, 재사용 가능하게 배포할 수 있도록 도와주는 도구입니다. 마치 apt(Ubuntu), brew(macOS), npm(Node.js) 같은 느낌.

Helm이 필요한 이유

Kubernetes에서 애플리케이션을 배포하려면 Deployment, Service, ConfigMap, Secret 등 여러 YAML 파일을 직접 작성하고 관리해야 합니다. 이게 좀 복잡하고 중복도 많아지는데… Helm은 이걸 템플릿으로 관리하면서 변수로 구성하고, 한 번에 배포/삭제/업데이트할 수 있게 해줍니다.

주요 개념

  • Chart: Helm의 패키지 단위. 하나의 애플리케이션(또는 마이크로서비스)을 구성하는 YAML 템플릿 모음
  • Release: Chart를 실제 클러스터에 설치한 인스턴스
  • Values: Chart에 넘길 변수값을 정의하는 파일 (values.yaml)
  • Templates: Deployment.yaml, Service.yaml 같은 Kubernetes 리소스를 Go 템플릿 형식으로 작성한 파일

Helm 사용 흐름 (간단 예시)

  1. Chart 설치 (배포):

    helm install my-app ./my-chart

  2. 업데이트:

    helm upgrade my-app ./my-chart

  3. 삭제:

    helm uninstall my-app

  4. 값 지정해서 설치:

    helm install my-app ./my-chart -f custom-values.yaml

Helm의 장점

  • 템플릿화: 반복적인 YAML 작성 최소화
  • 설정 분리: 환경에 따라 values 파일만 다르게 유지 가능
  • 버전 관리: 릴리즈 기록을 남기고 롤백 가능
  • 커뮤니티 Chart: Redis, MySQL, nginx 같은 유명 애플리케이션은 이미 공식 Helm Chart로 존재

helm chart 구조

아래는 간단한 웹 애플리케이션을 Kubernetes에 배포하기 위한 Helm Chart 기본 구조입니다. 이 예시는 my-app이라는 이름의 애플리케이션을 기준으로 설명드리겠습니다.

📁 Helm Chart 디렉토리 구조

my-app/
├── Chart.yaml
├── values.yaml
├── templates/
│   ├── deployment.yaml
│   ├── service.yaml
│   ├── ingress.yaml
│   └── _helpers.tpl

각 파일 설명

1. Chart.yaml

Helm Chart의 메타 정보를 정의하는 파일입니다.

apiVersion: v2
name: my-app
description: A simple web application
type: application
version: 0.1.0
appVersion: "1.0.0"

2. values.yaml

템플릿에서 사용할 기본 설정 값을 정의합니다. 이 파일은 환경별로 오버라이드할 수 있습니다.

replicaCount: 2

image:
  repository: mydockerhub/my-app
  tag: "1.0.0"
  pullPolicy: IfNotPresent

service:
  type: ClusterIP
  port: 80

ingress:
  enabled: false
  className: ""
  annotations: {}
  hosts:
    - host: my-app.example.com
      paths:
        - path: /
          pathType: Prefix
  tls: []

resources: {}

3. templates/deployment.yaml

Kubernetes의 Deployment 리소스를 정의합니다.

apiVersion: apps/v1
kind: Deployment
metadata:
  name: {{ .Release.Name }}-deployment
spec:
  replicas: {{ .Values.replicaCount }}
  selector:
    matchLabels:
      app: {{ .Release.Name }}
  template:
    metadata:
      labels:
        app: {{ .Release.Name }}
    spec:
      containers:
        - name: {{ .Release.Name }}
          image: "{{ .Values.image.repository }}:{{ .Values.image.tag }}"
          imagePullPolicy: {{ .Values.image.pullPolicy }}
          ports:
            - containerPort: 80

4. templates/service.yaml

서비스 리소스를 정의합니다.

apiVersion: v1
kind: Service
metadata:
  name: {{ .Release.Name }}-service
spec:
  type: {{ .Values.service.type }}
  selector:
    app: {{ .Release.Name }}
  ports:
    - port: {{ .Values.service.port }}
      targetPort: 80

5. templates/ingress.yaml (옵션)

Ingress를 사용하는 경우 정의합니다.

{{- if .Values.ingress.enabled }}
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: {{ .Release.Name }}-ingress
  annotations:
    {{- range $key, $value := .Values.ingress.annotations }}
    {{ $key }}: {{ $value | quote }}
    {{- end }}
spec:
  rules:
    {{- range .Values.ingress.hosts }}
    - host: {{ .host }}
      http:
        paths:
          {{- range .paths }}
          - path: {{ .path }}
            pathType: {{ .pathType }}
            backend:
              service:
                name: {{ $.Release.Name }}-service
                port:
                  number: {{ $.Values.service.port }}
          {{- end }}
    {{- end }}
{{- end }}

6. templates/_helpers.tpl

템플릿에서 재사용할 수 있는 함수 또는 변수 등을 정의하는 파일입니다. 선택 사항입니다.

이 구조를 기반으로 Helm Chart를 구성하시면, 다양한 환경에 따라 설정을 손쉽게 바꾸고, 재사용성과 유지보수성을 높일 수 있습니다.
더 복잡한 기능(예: ConfigMap, Secret, autoscaling 등)이 필요하시다면 해당 부분도 추가해서 확장하실 수 있습니다.
